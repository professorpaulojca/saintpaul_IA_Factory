# Engineering Standards

## Backend
- usar DTOs para entrada e saída
- tratar exceções de forma centralizada
- usar migrations versionadas
- documentar API com OpenAPI
- gerar testes unitários junto com cada service e controller

## Frontend
- manter separação entre serviço e componente
- evitar lógica de negócio na view
- tratar loading, erro e vazio
- gerar testes unitários junto com cada hook e serviço

## Banco
- garantir integridade referencial
- nomear objetos de forma consistente
- incluir índices quando fizer sentido

## Documentação
- diagramas devem refletir a especificação
- BPMN deve ser importável no Camunda sempre que possível
